## Prerequisites

Before you begin, make sure the following are in place:

- **Ubuntu** 18.04 (Melodic), 20.04 (Noetic / Foxy), or 22.04 (Humble)
- **ROS or ROS2** installed with Desktop-Full configuration
- **Livox-SDK2** installed — follow the [Livox-SDK2 installation guide](https://github.com/Livox-SDK/Livox-SDK2/blob/master/README.md)
- Your LiDAR connected over Ethernet and assigned an IP address on the same subnet as your host machine

<Tip>
  ROS2 builds use `colcon` as the build tool. If you don't have it installed, follow the [Colcon installation instructions](https://docs.ros.org/en/foxy/Tutorials/Beginner-Client-Libraries/Colcon-Tutorial.html).
</Tip>

## Steps

<Steps>
  <Step title="Clone the repository">
    Clone the driver source code into a `[work_space]/src/` folder. This structure is required by the build tooling.

    ```bash
    git clone https://github.com/Livox-SDK/livox_ros_driver2.git ws_livox/src/livox_ros_driver2
    ```

    <Warning>
      You must clone into a folder named `src/` inside your workspace (as shown above). Placing the source elsewhere causes compilation errors.
    </Warning>
  </Step>

  <Step title="Build the driver">
    Source your ROS environment, then run `build.sh` with the argument for your ROS version.

    <CodeGroup>

    ```bash ROS1 (Noetic)
    source /opt/ros/noetic/setup.sh
    ./build.sh ROS1
    ```

    ```bash ROS2 (Foxy)
    source /opt/ros/foxy/setup.sh
    ./build.sh ROS2
    ```

    ```bash ROS2 (Humble)
    source /opt/ros/humble/setup.sh
    ./build.sh humble
    ```

    </CodeGroup>

    The build script handles copying the correct `package.xml`, setting up the `launch/` directory for ROS2, and invoking either `catkin_make` (ROS1) or `colcon build` (ROS2).
  </Step>

  <Step title="Configure your LiDAR">
    The driver reads LiDAR settings from a JSON config file. The default config for HAP is at `config/HAP_config.json` and for Mid-360 at `config/MID360_config.json`.

    Open the config file for your device and update the IP addresses to match your network:

    ```json config/HAP_config.json
    {
      "lidar_summary_info": {
        "lidar_type": 8
      },
      "HAP": {
        "lidar_net_info": {
          "cmd_data_port": 56000,
          "push_msg_port": 0,
          "point_data_port": 57000,
          "imu_data_port": 58000,
          "log_data_port": 59000
        },
        "host_net_info": {
          "cmd_data_ip": "192.168.1.5",
          "cmd_data_port": 56000,
          "push_msg_ip": "",
          "push_msg_port": 0,
          "point_data_ip": "192.168.1.5",
          "point_data_port": 57000,
          "imu_data_ip": "192.168.1.5",
          "imu_data_port": 58000,
          "log_data_ip": "",
          "log_data_port": 59000
        }
      },
      "lidar_configs": [
        {
          "ip": "192.168.1.100",
          "pcl_data_type": 1,
          "pattern_mode": 0,
          "blind_spot_set": 50,
          "extrinsic_parameter": {
            "roll": 0.0,
            "pitch": 0.0,
            "yaw": 0.0,
            "x": 0,
            "y": 0,
            "z": 0
          }
        }
      ]
    }
    ```

    The key fields to update:

    | Field | Description |
    | ----- | ----------- |
    | `host_net_info.cmd_data_ip` / `point_data_ip` / `imu_data_ip` | IP address of your host machine |
    | `lidar_configs[].ip` | IP address of the LiDAR device |

    The launch file for each device points to its config via the `user_config_path` parameter. For HAP this is `config/HAP_config.json`; for Mid-360 it is `config/MID360_config.json`.
  </Step>

  <Step title="Source the workspace and launch the driver">
    Source the workspace setup file for your ROS version, then launch the driver with RViz to verify point cloud output.

    <CodeGroup>

    ```bash ROS1
    source ../../devel/setup.sh
    roslaunch livox_ros_driver2 rviz_HAP.launch
    ```

    ```bash ROS2
    source ../../install/setup.sh
    ros2 launch livox_ros_driver2 rviz_HAP_launch.py
    ```

    </CodeGroup>

    RViz will open automatically and display the incoming point cloud. If no points appear, check the **Global Options > Fixed Frame** field in RViz and set it to `livox_frame`, then enable the **PointCloud2** display.

    <Note>
      If you see a `liblivox_sdk_shared.so` error on ROS2, add `/usr/local/lib` to your library path:

      ```bash
      export LD_LIBRARY_PATH=${LD_LIBRARY_PATH}:/usr/local/lib
      ```

      To make this permanent, add the line to your `~/.bashrc` and run `source ~/.bashrc`.
    </Note>
  </Step>
</Steps>

## Available launch files

The `launch_ROS1/` and `launch_ROS2/` directories contain several preconfigured launch files:

| ROS1 launch file | ROS2 launch file | Device | Output |
| ---------------- | ---------------- | ------ | ------ |
| `rviz_HAP.launch` | `rviz_HAP_launch.py` | HAP | PointCloud2 + RViz |
| `msg_HAP.launch` | `msg_HAP_launch.py` | HAP | Livox customized point cloud |
| `rviz_MID360.launch` | `rviz_MID360_launch.py` | Mid-360 | PointCloud2 + RViz |
| `msg_MID360.launch` | `msg_MID360_launch.py` | Mid-360 | Livox customized point cloud |
| `rviz_mixed.launch` | `rviz_mixed.py` | HAP + Mid-360 | PointCloud2 + RViz |
| `msg_mixed.launch` | — | HAP + Mid-360 | Livox customized point cloud |
